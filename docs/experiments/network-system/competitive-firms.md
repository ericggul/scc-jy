# Network System / Competitive Firms

## 현재 이터레이션의 결정

이 실험에서 **상위 레벨**은 컨트롤러의 네 회사 경쟁망이고, **하위 레벨**은 각 회사 스크린의 경영 시스템이다. 네 회사는 동일한 층위에 있으며 같은 상태·정책·방정식을 갖는다. 현재 이터레이션에서 상위 레벨은 읽기 전용이다. 시장점유율이나 회사 사이 관계를 사용자가 직접 쓰지 않는다.

하위 레벨에서 이루어진 경영 결정이 상품·가격·공급능력·자본을 바꾸고, 그 결과 고객 이동과 시장점유율이 달라진다. 달라진 수요와 판매가 다시 각 회사의 자본과 공급능력에 돌아온다.

```text
하위 경영정책 → 회사 상태 → 상대적 고객 효용 → 회사 간 고객 이동
      ↑                                      ↓
      └──── 자본·공급·판매 ← 수요·시장점유율 ────┘
```

이것은 정확한 기업 예측 모델이 아니라, 상·하위 그래프 사이의 양방향 동역학을 경험하기 위한 절제된 실험 모델이다. 임의의 상위 가중치나 탑다운 시장점유율 조절은 없다.

## 하위 레벨: 노드 단위 경영

각 회사 화면은 동일한 네 노드를 갖는다. 노드 중앙의 큰 값은 실제 회사 상태이고, 아래 `−/+`는 상태가 아닌 지속적인 경영 목표를 조절한다.

| 노드 | 조절 정책 | 지연·비용·상충관계 |
|---|---|---|
| Product | R&D intensity | 자본을 소모하며 상품성을 지연 상승시킨다. R&D 생산성에는 체감효과가 있고 상품성은 유지비 없이 영구 보존되지 않는다. |
| Price | target price | 실제 가격이 목표로 접근한다. 가격 인하는 고객 효용을 높일 수 있지만 단위 마진을 낮춘다. |
| Capacity | target capacity | 공급능력이 목표로 점진 이동한다. 확장은 가용 자본에 제한되고 확장비와 고정비가 든다. |
| Capital | leverage target | 부채 조달은 단기 자본을 늘리지만 이자 비용을 지속 발생시킨다. 축소는 상환으로 자본을 사용한다. |

따라서 버튼은 `Product +0.1` 같은 직접 상태 충격이 아니다. 사용자가 정하는 것은 회사가 앞으로 좇을 정책이고, 시스템이 그 결과를 시간에 따라 계산한다.

## 상위 레벨: 고객 이동 네트워크

상위의 12개 유향 엣지 `i → j`는 회사가 회사에 가하는 임의 압력이 아니라 **회사 i의 고객 중 회사 j로 이동하는 비율 흐름**이다. 고객은 세 항목을 상대적으로 평가한다.

- 상품성
- 가격
- 현재 수요를 실제로 충족할 수 있는 공급 가능성

상품성의 고객 효용은 무제한 선형 증가하지 않고 체감한다. 가격 불이익은 상대 가격비에 따라 작동하므로, R&D로 먼저 우위를 만든 회사라도 가격을 대폭 올리면 고객 이동이 다시 발생한다. 각 출발 회사에서 잔류 선택과 세 대안 회사의 효용 차이를 함께 비교해 이동량을 만든다. 모든 유출은 다른 회사의 유입이므로 시장점유율 합은 1로 보존된다. 이동량은 출발 회사 점유율에 비례하지만 유입은 목적 회사의 기존 점유율에 의존하지 않으므로, 0% 회사도 더 나은 조건을 만들면 회복할 수 있다.

## 상위 인터페이스의 두 관측 방식

우측 상단 스위치는 시스템 개입이 아니라 같은 상태를 보는 로컬 뷰 전환이다.

- `NETWORK`: 네 회사, 12개 고객 이동 방향, 이동량, 결과 시장점유율을 G(V,E)로 본다.
- `MARKET`: `stock/1`의 검은 카드·타이포그래피·상승/하락 색·차트 문법을 재사용해 네 회사 시장점유율 이력을 2×2로 본다.

컨트롤러에는 엣지 가중치 버튼, 회사 충격 버튼, 리셋 버튼이 없다.

## 실제 내부 엣지만 표시

하위 화면은 K4를 장식적으로 완전 연결하지 않는다. 서버 방정식에 실제로 쓰이는 다음 인과만 그린다.

- Capital → Product: R&D가 만드는 상품 성장
- Product → Capital: R&D 비용
- Product → Price: 상품성이 주는 가격 여력
- Capital → Capacity: 자본 제약 아래의 확장
- Capacity → Price: 수요 대비 부족/여유
- Capacity → Capital: 가변비, 고정비, 확장비
- Price → Capital: 판매 수입

## 화면 계약

단일 스크린은 특정 종횡비를 가정하지 않고 실제 viewport에 반응한다. `whole`에서는 같은 전체 화면 컴포지션을 `200% × 200%` 캔버스에 렌더한 뒤 `scale(0.5)` 하므로, 2×2 안에서 재배치된 별도 디자인이 아니라 단일 화면이 그대로 축소되어 보인다.
