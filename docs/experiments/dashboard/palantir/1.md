# Global situation monitor experiment

## Corrected target

The reference is not a generic Palantir or industrial-finance dashboard. It is
a World Monitor–style global situational-awareness surface: a map-first OSINT
workstation combining conflict and infrastructure layers, public event feeds,
live news video, live webcams, and source-attributed strategic summaries.

The public route remains `/palantir/1`, but the experiment itself follows the
reference image's global-operations context.

## Interface contract derived from the reference

```text
┌────────────────────── compact global operations toolbar ────────────────────────┐
├──────────────────────── GLOBAL SITUATION + UTC ──────────────────────────────────┤
│ ┌ time + layers ┐                  world map                    ┌ map controls ┐ │
│ │ scenario      │  conflict / intel / base / nuclear / API     │ 2D 3D + −   │ │
│ │ infrastructure│  dots, routes, zones, labels, selection       └──────────────┘ │
│ └───────────────┘                                                                │
├──────────────────────────┬──────────────────────────┬─────────────────────────────┤
│ live news + switcher     │ live webcams + regions  │ sourced brief + posture    │
│ one large moving stream  │ four moving streams     │ theater assessments        │
└──────────────────────────┴──────────────────────────┴─────────────────────────────┘
```

Unmentioned visual properties of the reference are treated as invariants:
near-black surfaces, thin functional borders, compact mono typography, dense
controls, no rounded card dashboard, a dominant world map, a floating left
layer stack, and a 40/40/20 lower media grid.

Do not add a faux browser, operating-system window, capture bar, traffic-light
buttons, product-status strip, or other decorative frame above the functional
application toolbar. The user explicitly rejected that topmost UI. This
experiment begins with its real application navigation.

## Typography and responsive contract

This experiment follows the legibility lesson preserved in
[`stock/3`](../stock/3.md): information density must come from
complete records and useful layers, not from globally tiny type.

- The shared desktop type scale is
  `clamp(12px, min(0.78vw, 1.3vh), 15px)` using SF Mono–compatible faces.
- Functional controls, layer rows, media tabs, panel headings, map legends, and
  strategic summaries have explicit `9–11px` minimums instead of compounding
  sub-`em` sizes down to unreadable values.
- Above 980px the workstation fills `100dvh`, with the world map and lower
  40/40/20 media region sharing all space remaining below the two functional
  top bars.
- From 721–980px the media region becomes two equal columns and the strategic
  panel spans the full row. The document can scroll so content is not clipped.
- At 720px and below, news, webcams, and strategic content become a single
  column. The map keeps a deliberate fixed working height and its layer panel
  remains inspectable rather than being hidden.
- At 430px and below, secondary toolbar metadata is removed before primary
  controls are compressed; the map popup moves above the lower controls and
  the media panels retain their content heights.

Changing record count or adding another map control requires checking both the
desktop allocation and every reflow height. Shrinking the root font to recover
space is not an acceptable fix.

## Data and media sources

- Country geometry: the `countries.geojson` dataset shipped by the public
  World Monitor repository, based on Natural Earth geography.
- Dynamic earthquakes: USGS all-day GeoJSON feed, filtered to magnitude 2.5+.
- Dynamic natural events: NASA EONET open-events API.
- Live news: ten current channels covering Bloomberg, Sky News, Euronews, DW,
  CNBC, France 24, Al Jazeera, CBS News, TRT World, and Al Arabiya. Nine have
  current direct HLS manifests and all ten have current YouTube fallbacks. Native HLS
  is preferred when supported, `hls.js` handles Chrome/Firefox, and YouTube is
  the automatic fallback. The configured HLS manifests were checked for valid
  `#EXTM3U` responses on 2026-07-23.
- Live webcams: current YouTube streams for Jerusalem, the Middle East, Kyiv,
  Washington, Tel Aviv, Beirut, Mecca, London, Odessa, Paris, St. Petersburg,
  New York, Los Angeles, Miami, Taipei, Tokyo, Shanghai, Sydney, and two ISS
  views. The feeds were checked against YouTube oEmbed metadata on 2026-07-23;
  one stale Seoul ID was removed and the current official Jerusalem EarthCam
  ID replaced the stale repository value.
- Static operational reference points: known conflict theaters, military
  facilities, IAEA-relevant nuclear sites, spaceports, hyperscale data-center
  corridors, strategic pipelines, and undersea cable routes. Static points are
  described as reference data; the Iran layer is explicitly a simulation.
- The 23 July 2026 brief uses current AP, Reuters, and Al Jazeera reporting
  discovered during implementation. It is source-labelled and is not presented
  as a continuously generated AI result.

The source implementation states an important webcam limitation: Windy webcam
players are usually timelapses built from still captures, not live video. This
experiment therefore uses the curated YouTube live-video feeds instead.

## Interaction and state

- Layer checkboxes change the visible map data.
- The six time-window controls preserve the reference interaction grammar.
- 2D/3D, wheel zoom, drag pan, reset, and Middle East focus controls change map
  framing. The map viewBox follows the rendered container aspect ratio so the
  Web Mercator geometry fills the panel without being stretched. The projection
  uses a square `1000 × 1000` world coordinate plane; using a `1000 × 500`
  Mercator plane vertically compresses every landmass and is prohibited.
- Map markers expose their source type, severity, coordinates, and description.
- News channel buttons switch one persistent moving stream; play/pause is
  explicit.
- Webcam region tabs swap a four-stream wall.
- USGS and NASA feed status is visible and falls back safely when a public API
  cannot be reached.

## External reference

for this repository rather than copied as a component transplant. Public data
